What is Claude, and why are we using it?

If you've never used an AI assistant before — or if you've tried one and weren't sure what to do with it — this section is for you.

Claude is a brilliant, patient AI assistant you talk to in plain language.

You type what you need — a question, a task, a document to analyze — and Claude responds. No special commands, no technical skills required. If you can write an email, you can use Claude.

🧠
It understands context

You can have a back-and-forth conversation. Claude remembers what you said earlier in the same chat and builds on it.

📎
It reads your files

Attach a PDF, a Word doc, a spreadsheet, or an image. Claude reads it and works with the content — no copy-pasting needed.

🌐
It searches the web

When you need current information, Claude can search online in real time and bring you a synthesized, cited answer.

Why is EyeSee rolling this out now?

EyeSee's work — research, analysis, client communication, proposals, reporting — is exactly the kind of knowledge work where AI creates the most time savings. The goal is not to replace how we work. It's to remove the slow, repetitive parts so each of us can focus on the thinking that actually requires our expertise.

The three things Claude is genuinely great at — at EyeSee

1

First drafts. Any document that follows a pattern — reports, emails, proposals, SOPs, summaries — Claude produces a strong starting draft in seconds. You refine it; Claude does the heavy lifting.

2

Making sense of data and documents. Drop in a file and ask questions. "What are the key takeaways?" "Which numbers look unusual?" "Summarize this for a non-technical audience." Claude reads it and responds.

3

Research and synthesis. Instead of spending an hour reading 10 websites, describe what you need and Claude searches, reads, and delivers a structured summary with sources — in under a minute.

!

One important rule: Claude can occasionally be wrong — especially on specific facts and numbers. Always review before sending anything to a client or using in a decision. Treat it as a very capable colleague, not an oracle.

How to talk to Claude — the only skill you need to learn

You don't need to learn special commands or formats. Claude understands plain language. But the way you phrase a request significantly affects the quality of the answer. One principle covers 90% of it: give context, not just a command.

❌ Weak prompt
"Write a follow-up email."
Claude has no idea who it's to, what it's following up on, what tone you want, or what you need the recipient to do. You'll get a generic template.
✅ Strong prompt
"Write a short follow-up email to our Belgian client Unilever. We met last Tuesday to discuss a new pack testing project. They haven't responded in 5 days. Tone: warm but nudging. Keep it under 80 words."
Claude has who, what, why, tone, and length. The output will be immediately usable.

Six things that make any Claude prompt better

  • Say who it's for and why. "Draft a message to my direct report" hits differently than "Draft a message."
  • Specify the format. "Give me 3 bullet points" or "Write this as a formal memo" or "Make it a table." Claude defaults to prose otherwise.
  • Give a length target. "Under 100 words" or "one paragraph" or "a 10-slide outline." Claude tends to over-produce without a limit.
  • Assign a role when useful. "You are an experienced management consultant" or "Act as a tough editor who cuts ruthlessly." It shifts the register.
  • Attach the actual files. If you have a document that's relevant — attach it. Don't describe what's in it. Claude reads it directly.
  • Iterate out loud. If the first output isn't right, say so directly: "Too formal — make it warmer." "Cut the second paragraph." "Add a section on pricing." You're in a conversation.

Skills

Skills are the most powerful — and least known — feature in Claude. Once set up, they eliminate the need to re-explain the same task over and over. You just work, and Claude automatically knows what to do and how.

The core idea

A Skill is a SKILL.md file — a structured text file containing instructions for a specific task or workflow. You install it into Claude once. After that, Claude reads it automatically whenever you're doing that type of task. You don't invoke a Skill. It invokes itself. Claude recognizes what you're working on and activates the right Skill on its own.

Skills vs. regular prompts vs. Projects

ApproachHow it worksLimitation
Long prompt You write all the context and instructions every time you start a task You repeat yourself constantly. Exhausting.
Uploaded file You keep instructions in a doc and attach it to conversations You have to remember to attach it and say "read this first" every time
Project Instructions and docs are pre-loaded in a specific Project workspace You have to open the right Project. Context is fixed per Project.
Skill ✦ A SKILL.md file installed in Claude. Fires automatically when the task matches. Needs initial setup. Description quality matters.

What a Skill contains

🎯 A trigger description

Tells Claude when to activate this Skill. Example: "Use when the user asks to write a client report or summarize research findings." Also defines when NOT to use it.

📋 Step-by-step instructions

The full workflow for the task: what format to use, what sections to always include, what tone to adopt, what to check before finishing. The more specific, the better.

⚡ Slash command (optional)

You can also call any Skill manually by typing a command like /report or /proposal. Useful when you want to force it rather than wait for auto-detection.

🔗 Stacks with everything else

Skills work on top of Projects and uploaded files. Your Skill handles the process. Your Project documents provide the context. They fire together, simultaneously.

Two ways to create a Skill

🛠️ Option A — Claude's Skill Creator (recommended)

Open Claude Cowork → type "Use the skill-creator to help me build a skill for [your task]." Claude interviews you, generates the SKILL.md file, and runs an evaluation to test it before you install. Takes about 15 minutes for your first skill.

⚡ Option B — makemyskill.com (faster)

Go to makemyskill.com → describe the task in a paragraph → it searches the web and writes the Skill for you. Faster, less control. Good for simpler Skills or if you want to skip the interview. Download and upload to Claude.

How to install a Skill

1

Create or download the SKILL.md file

Use the Skill Creator in Cowork, makemyskill.com, or browse Anthropic's pre-built Skills inside Cowork → Customize → Personal Plugins → Browse.

2

Install it: Settings → Capabilities → Skills → Upload

In Claude → left menu → Customize → Skills → the "+" button → Upload a skill. The SKILL.md file goes in a named folder.

3

Test it with 5 different phrasings

Type requests that should trigger the Skill. Then try unrelated requests to make sure it stays quiet when it shouldn't fire. If something is off, ask Claude: "When would you use this skill?" — it quotes the trigger description back to you so you can fix it.

4

Share with your team

Skills can be shared. Once a head of department has a working Skill, it can be distributed to the whole team — everyone gets the same consistent workflow with no individual setup.

Two things that make the difference between a good Skill and a useless one

  • "Do NOT use for…" matters more than "Use when…" A Skill with no negative triggers will hijack conversations it shouldn't. Always define what the Skill is NOT for, as specifically as what it IS for.
  • Specificity beats length. "I write reports" is useless. "I write weekly client reports that always start with the headline finding, use 3 insight sections, and end with next steps as bullet points" is a Skill that works the first time.

Connectors & MCP

Connectors let Claude reach directly into the tools you already use — reading files, pulling data, and taking actions — without you copying and pasting anything. Powered by an open standard called MCP (Model Context Protocol), they work across Claude Chat, Cowork, and Claude Code.

🔌
What MCP is

A universal plug standard. Instead of uploading files, Claude gets live, secure access to your tools. It reads what's there right now — not a copy from last week.

🔐
Your permissions, respected

Claude inherits your existing access rights. If you can't see a file in SharePoint, Claude can't either. No new permissions are granted — it works within your existing security model.

🆓
Free to add

All connectors are free. One-time setup, then they're available in every conversation. Browse the full library at claude.com/connectors.

Data & Privacy
🔒

Is your data safe?

A common question when rolling out any AI tool. The short answer for EyeSee's Team plan: yes. Here's exactly what Anthropic's official policy says and what it means in practice.

Official Anthropic policy — Team plan (Commercial Terms)

  • No training on your data. Anthropic does not use your prompts, files, or conversations to train AI models. This is guaranteed under the Commercial Terms of Service that covers the Team plan.
  • No selling your data. Anthropic does not sell user data to third parties. This applies to all plans including Team.
  • 30-day data retention. Anthropic retains conversation data on their servers for 30 days, after which it is deleted. You can delete individual conversations at any time.
  • The training opt-in only applies to consumer plans. The 2025 policy change that allows data training only affects Free, Pro, and Max individual accounts — it explicitly excludes Team and Enterprise plans.
✅ Safe to use

Financial reports, budget analyses, cost tables, internal strategy documents, HR materials (job descriptions, onboarding docs, policy drafts), M&A research notes, client research summaries, competitive briefs.

⚠️ Apply judgment

Individual employee personal data (salaries, health info, addresses) — not because Claude trains on it, but as general data hygiene. Client PII or data covered by client contracts — check whether those contracts permit sharing with AI tools.

If you ever need zero retention (Enterprise): Anthropic offers a Zero Data Retention (ZDR) option on the Enterprise tier — where conversation data is never stored on Anthropic's servers at all. This is relevant if EyeSee starts handling highly regulated client data at scale or needs to meet specific compliance requirements. Ask Novak if this becomes relevant.

Three rules to follow regardless

  • Never paste passwords, API keys, or payment credentials into any AI tool — this is a universal rule, not Claude-specific.
  • Check client contracts before sharing client-specific raw data — some enterprise agreements have restrictions on third-party AI processing.
  • Don't share individual employee personal data (salaries, performance reviews, medical info) unless it's anonymized or aggregated.

Initial Setup

Before you start using Claude for real work, spend 30 minutes setting it up properly. This is the difference between Claude feeling like a generic chatbot and feeling like a tool that actually knows you and your work.

1

Create your about-me.md — your context file

A markdown text file that tells Claude who you are, what you do, and how you like to work. Load it into a Project once. Claude reads it before every conversation, so you never re-explain yourself. Include: your role at EyeSee, your team, the types of work you do most, your communication style preferences, and anything Claude should always know (e.g. "I work with FMCG clients" or "I prefer concise, direct language"). 1–2 paragraphs is enough to start.

I am [Name], Head of [Dept] at EyeSee — a behavioral market research company based in Antwerp. My team of [X] people handles [core function]. I work primarily with [clients / stakeholders]. I prefer responses that are direct and concise. When drafting communications, always use a professional but warm tone. My most common tasks are [list 3-4 things].

5 things to try this week

If you're not sure where to start, try these five. Each one is useful on its own and teaches you something about how Claude works.

1

Summarize your last messy meeting

Copy your last set of meeting notes — however rough — and paste them into Claude. Say: "Extract all decisions made and action items with owners. Format as a clean summary I can send to the team."

What you'll learn: Claude turns raw input into structured output instantly. This alone saves 20 minutes per meeting.
2

Get a first draft of something you write often

Think of one type of document you write repeatedly — a project update, a client email, a weekly report. Give Claude the context and ask for a draft. Then say what's wrong with it and ask for a revised version.

What you'll learn: Claude's first draft + one correction cycle is faster than writing from scratch. Every time.
3

Research a company or competitor in 2 minutes

Enable web search in Claude settings, then ask: "Research [company name]. Give me a 1-page summary: what they do, size, key products, recent news, and whether they compete with or complement EyeSee." Compare to how long a manual Google search would take.

What you'll learn: Claude doesn't just find links — it reads, synthesizes, and gives you a structured answer with sources.
4

Attach a document and have a conversation with it

Attach any PDF, Word doc, or Excel file you've been meaning to read. Ask: "What are the 5 most important points?" Then follow up: "What would you recommend we do based on this?" Then: "Are there any risks or gaps not addressed here?"

What you'll learn: Claude reads and reasons about your files. You can interrogate a document like a colleague who's read it thoroughly.
5

Use Extended Thinking on a real strategic question

Switch to Opus 4.6, enable Extended Thinking, then ask something genuinely complex: a decision you're wrestling with, a strategy question, a problem with competing priorities. Give it full context. Watch it think through the problem before answering — the output will surprise you.

What you'll learn: This is not autocomplete. Opus 4.6 with Extended Thinking reasons like a senior advisor. Use it for the hard problems.

Claude Dictionary

Every term you'll encounter in Claude, explained in plain language. No technical background needed.

● Core Products ● Features ● Files & Config ● Concepts
Core Product

Claude Chat

The web and mobile interface at claude.ai. Your main AI assistant. Best for drafting, analysis, research, and strategic thinking. Where you'll spend most of your time.

Core Product

Claude Cowork

Desktop app where Claude operates your computer and browser. Think of it as Claude with hands — it browses websites, reads your files, and executes multi-step tasks while you watch.

Core Product

Claude Code

Terminal-based tool for writing and running code. Used by R&D developers. Not for heads of department — but it's how R&D builds the tools you request.

Feature

Projects

A workspace inside Claude Chat where you load instructions and documents once. Every conversation inside the Project starts with that context already active. Shareable with your team.

Feature

Extended Thinking

A mode where Claude reasons step-by-step before answering. Slower, but significantly smarter on complex problems. Always pair it with Opus 4.6 — the combination is Claude at its best.

Feature

Connectors

Integrations that give Claude live access to your tools (M365, Slack, Salesforce). Claude reads directly from those systems in real time — no file uploads needed. Powered by MCP.

Feature

Skills

SKILL.md files that encode a workflow and install it into Claude. They fire automatically when Claude recognizes the task — no command needed. Think: trained reflexes for repeated work.

Feature

Plugins

Pre-built bundles of Skills + Connectors packaged for a specific job function (e.g. Financial Analysis, HR, Sales). Connectors = download from the App Store. Plugins = upload your own app.

Feature

Skill Creator

Claude's built-in tool for building Skills. Open Cowork, ask it to "use the skill-creator", answer its interview, and it generates a ready-to-install SKILL.md file. No technical knowledge needed.

Feature

Scheduled Tasks

Set Claude to run a task automatically at a chosen frequency — daily, weekly, whenever. Pick the task, set the schedule, Claude runs it while your computer is on. Good for recurring reports or summaries.

Feature

Claude Computer

Claude's ability to click, browse, and operate your desktop while you're away. Available inside Cowork. Tell it what to do; it executes — opening tabs, filling forms, navigating sites autonomously.

Feature

Scoped Memory

Memory that lives inside a specific Project. You say "build on last week's report" and Claude knows what that is — but only inside that Project. Conversations outside don't share this memory.

Files & Config

SKILL.md

The file inside every Skill folder. Contains: the trigger description (when to use it), the /command name, and the step-by-step instructions for the task. One file = one installed capability.

Files & Config

about-me.md

A markdown file describing who you are, what you do, and how you work. Load it into your Project once. Claude reads it before every conversation — so you never re-explain your role or preferences. One file is worth 50 repeated prompts.

Files & Config

Context Files

Any document you upload to a Project as background knowledge — templates, guidelines, service descriptions, process docs. The more relevant context you give Claude, the less prompting you need each time.

Files & Config

Global Instructions

A persistent prompt that Cowork reads before every single task — across all sessions. Set it once, it runs forever. Use it for standing rules: preferred formats, tone, always-on context about you or your company.

Concept

Opus 4.6

Claude's most powerful model. Slower and uses more of your plan's capacity — but significantly smarter on hard problems. Always use it with Extended Thinking for complex, high-stakes tasks.

Concept

Negative Triggers

The "Do NOT use for…" lines in a Skill's description. More important than the positive triggers — they prevent your Skill from activating in conversations it shouldn't touch. Always define them.

Concept

MCP

Model Context Protocol. The open standard Anthropic built to connect Claude to external tools. Think of it as a universal plug — any app that supports MCP can be connected to Claude. Powers all Connectors.
